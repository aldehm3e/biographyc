# Biography CMS Database Guide

Author: Eng. Abdulrahman alsaedi

This guide explains how the PHP/MySQL part of Biography CMS works and how to keep installation, backup, restore, and deployment safe.

For a simple first-time setup, read `README_INSTALL_BEGINNER.md`.

## Storage Model

After installation, MySQL/MariaDB is the source of truth.

The browser loads content from:

```text
GET api/content/get-site.php
```

The admin panel saves content through:

```text
POST api/content/save-site.php
```

Runtime configuration is stored in:

```text
api/config.php
```

`api/config.php` is generated by the installer and must not be committed to Git.

## Important Files

- `install/index.php`: browser installer.
- `install/schema.sql`: main database schema.
- `api/install/schema.sql`: API installer schema copy.
- `api/install/seed.php`: initial seed helpers.
- `api/db.php`: PDO, session, auth, JSON, permission, and helper functions.
- `api/content/site-repository.php`: maps database rows to the frontend site data model.
- `api/content/get-site.php`: public read endpoint.
- `api/content/save-site.php`: admin-only save endpoint.
- `api/auth/`: login, logout, current user, user management, and account updates.
- `api/upload/upload-media.php`: admin-only uploads.

## Main Tables

The schema creates tables for:

- admin users
- site settings
- navigation items
- main page content
- hero slides
- experiences
- achievements
- skills
- projects
- pages
- contacts
- footer links
- media uploads
- integrations
- notifications
- backups or supporting CMS data

Footer structure and interface text overrides are stored as JSON fields in `site_settings`.

## Page Data Rules

Pages support:

- title
- slug
- one-level parent/subpage relation
- text or HTML content mode
- content body
- image path
- video path
- visible status
- header visibility
- footer visibility

Only root pages may appear as main header items. Subpages must stay under their parent page dropdown.

The repository also normalizes malformed page data so nested/cyclic subpages do not break navigation.

## Local Laragon Database

Normal database name:

```text
biography_cms
```

Create it:

```powershell
mysql -u root -e "CREATE DATABASE biography_cms CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

Installer values for a default Laragon setup:

```text
DB host: localhost
DB name: biography_cms
DB user: root
DB password: empty unless you set one
```

Open:

```text
http://localhost/Biography/install/
```

## Installation Output

The installer creates:

```text
api/config.php
install/install.lock
```

`install/install.lock` blocks accidental reinstall after setup.

## Reinstall Safely

Only reinstall when you intentionally want a fresh setup.

1. Back up the database and `uploads/` if needed.
2. Remove:

```text
api/config.php
install/install.lock
```

3. Drop or empty the old database, or create a new database.
4. Open:

```text
http://localhost/Biography/install/
```

Do not remove `uploads/` unless you also want to remove uploaded files.

## Production Deployment

Upload the project to `public_html` or the correct subfolder.

Protect these production files and folders during future updates:

```text
api/config.php
install/install.lock
uploads/
```

Production hosting must provide:

- PHP 8+
- PDO
- PDO MySQL
- sessions
- writable upload folders
- MySQL or MariaDB

Writable folders:

```text
uploads/images/
uploads/video/
uploads/logos/
uploads/icons/
```

## Backup

Back up both:

```text
MySQL database
uploads/
```

The database stores text, settings, and media paths. The actual uploaded files live in `uploads/`.

## Restore

1. Import the SQL database backup.
2. Restore the matching `uploads/` folder.
3. Confirm `api/config.php` points to the restored database.
4. Open `admin.html` and check that content and media both load.

## Schema Changes

When changing schema:

- Update `install/schema.sql`.
- Update `api/install/schema.sql` if it is kept as a separate installer copy.
- Add safe runtime compatibility in `api/content/site-repository.php` if existing installs need new columns.
- Test a fresh import into a temporary database.
- Test save/fetch through the repository.

## PHP Repository Notes

`api/content/site-repository.php` should remain conservative:

- Use prepared statements.
- Normalize all incoming admin data.
- Keep page/subpage rules server-side, not only in JavaScript.
- Keep schema auto-checks outside active transactions.
- Avoid repeated expensive schema checks per request.
- Keep Arabic strings UTF-8 safe.
- Treat the `utilities` permission as a trusted backup/restore permission because it can import or reset the full site data model.

`api/db.php` contains shared helpers. String truncation must preserve valid UTF-8.

## QA Commands

PHP syntax:

```powershell
$php = "C:\laragon\bin\php\php-8.3.30-Win32-vs16-x64\php.exe"
Get-ChildItem -Recurse -Filter *.php |
  Where-Object { $_.FullName -notmatch '\\(vendor|node_modules|backup|backups)\\' } |
  ForEach-Object { & $php -l $_.FullName }
```

JS syntax:

```powershell
Get-ChildItem -Recurse -Filter *.js |
  Where-Object { $_.FullName -notmatch '\\(vendor|node_modules|backup|backups)\\' } |
  ForEach-Object { node --check $_.FullName }
```

Fresh schema import:

```powershell
mysql -u root -e "CREATE DATABASE biography_qa CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
mysql -u root biography_qa < install/schema.sql
mysql -u root -e "DROP DATABASE biography_qa;"
```

Repeat with `api/install/schema.sql` when that file changes.

## Troubleshooting

Installer cannot connect:

- Check DB host, name, user, and password.
- Confirm MySQL is running.
- Confirm the database exists.

Installer says PDO MySQL is missing:

- Enable the PHP `pdo_mysql` extension.
- Use a PHP build with MySQL support.

Admin cannot save:

- Confirm login works.
- Confirm the user has permission for that section.
- Confirm `api/config.php` is correct.
- Check PHP error logs.

Uploads fail:

- Confirm the upload folder exists.
- Confirm the folder is writable.
- Confirm file size/type is allowed.

Arabic text looks broken in terminal:

- First check the browser and file encoding.
- Windows console code pages can display UTF-8 Arabic incorrectly even when the file is fine.
